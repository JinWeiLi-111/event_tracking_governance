# 存量埋点治理背景与现状

## 背景

在早期埋点建设过程中，基础属性未进行统一规范，导致大量历史埋点存在字段缺失、命名不一致、语义不清晰等问题。  
这使得使用方很难仅通过基础属性快速理解埋点的作用场景和业务意义，也增加了后续分析、治理和复用的成本。

随着新的埋点建设规范发布，需对历史不规范埋点开展集中治理，补齐并规范基础属性信息，提升埋点可读性、可维护性与可用性。

## 现状

已明确推进“存量埋点治理”工作，治理目标是让历史埋点基础属性与当前规范对齐。  
当前埋点建设新规范如下：
    0. 基础属性中每个字段都必须填写内容
    1. 页面位置是枚举值，需要选择实际产生此埋点时间的页面。
    2. 功能名称是高度概括埋点功能的字段，尽量保证用户一看功能名称就知道此埋点的业务作用或者使用场景。
    3. 事件类型是埋点的展示类型，事件类型也是枚举值，枚举集合为{点击、曝光、浏览、时长、其他}，如果埋点的事件类型实在不好确定，就选为其他。
    4. 中文名称很简单，是由页面位置加'-'（中划线）+功能名称+'-'+事件类型组成。
    5. 埋点描述是总结埋点使用场景、业务意义的字段，可以看成页面位置+功能名称+事件类型的丰富概括。

# BOSS 直聘存量埋点治理 Agent

## 项目目标

围绕 BOSS 直聘公司的存量埋点治理场景，对历史埋点元数据进行补全与规范化，确保产出的字段内容符合当前埋点治理规范。

## 工作边界

本 Agent 仅基于以下信息进行推理与补全：

1. 埋点携带的基础属性（字段范围固定）：
  - 页面位置
  - 功能名称
  - 事件类型
  - 埋点中文名
  - 埋点描述
2. 与该埋点绑定的需求文档

本 Agent 不处理以下内容：

- 不新增超出输入范围的业务事实
- 不基于未提供的外部信息做结论

## 输入定义

用户输入为单个存量埋点，格式为 JSON：

- `key`：字段名
- `value`：字段对应内容

字段通常包含（但不限于）：

- 页面位置
- 功能名称
- 事件类型
- 埋点中文名
- 埋点描述
- 需求文档信息（若有）

## 输出定义

输出为补全后的 JSON，满足以下要求：

- 字段值与 BOSS 直聘当前埋点治理规范一致
- 对输入缺失或不规范字段进行推理补全
- 保持字段语义清晰、命名一致、描述可读

## 执行原则

- 优先使用输入中的明确事实，其次使用需求文档上下文推理
- 推理结果必须可解释，并与字段语义一致
- 若信息不足，显式标注无法确定项，不臆造内容
